
# Food Fleet - Application Blueprint

## Overview

Food Fleet is a modern, visually appealing food delivery application designed to provide a seamless user experience. It features a clean, intuitive interface with a focus on usability and performance. The application is built with Next.js and utilizes a component-based architecture with a global state management for the shopping cart.

## Style and Design

- **Color Theme:** The application uses a vibrant and energetic red color theme. The primary color is `#dc2626`, used for buttons, links, and other interactive elements.
- **Typography:** A clean and readable font is used throughout the application, with a clear hierarchy for headings and body text.
- **Layout:** The layout is designed to be responsive and mobile-friendly, with a focus on clean spacing and a balanced visual hierarchy.
- **Iconography:** Modern and intuitive icons are used to enhance the user experience.
- **Visual Effects:** Subtle visual effects, such as shadows and gradients, are used to create a sense of depth and tactility.

## Features

- **Global Header:** A consistent header is present across all pages, providing navigation and access to the shopping cart.
- **Shopping Cart:** The application features a global shopping cart that allows users to add, remove, and update the quantity of items. The cart state is managed globally and is accessible from all pages.
- **Cart Page:** A dedicated cart page displays the contents of the shopping cart, allowing users to review their order before proceeding to checkout. The cart page is now fully responsive and optimized for mobile devices.
- **Order Type Selection:** On the cart page, users can select their preferred order type: "Take Away" or "Dine In."
- **Takeaway Scheduling:** When "Take Away" is selected, users can schedule their pickup by selecting a date and time.
- **Car Plate for Pickup (Optional):** For takeaway orders, users can optionally provide their car plate number to facilitate a quicker curbside pickup.
- **Dine-In Reservation:** When "Dine In" is selected, users can reserve a table by selecting a specific date and time.
- **Order Confirmation Dialog:** After proceeding to checkout, a confirmation dialog appears, displaying a comprehensive summary of the order, including order ID, date, payment status, total items, and a detailed list of items. The cart is automatically cleared after the user closes the dialog.
- **Shop Detail Page:** A detailed page for each shop, displaying the shop's information, menu, and a list of products. The shop detail page is now fully responsive and optimized for mobile devices.
- **Marketplace Landing Page:** A visually engaging landing page that showcases featured shops and products. The marketplace page is now fully responsive and optimized for mobile devices.
- **Responsive Design:** The application is fully responsive and adapts to different screen sizes, providing a seamless experience on both mobile and desktop devices.
